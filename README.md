Система учета лекарственных препаратов

Консольное приложение на Python для управления каталогом лекарственных средств с использованием базы данных PostgreSQL. 
Реализует полноценный CRUD (Create, Read, Update, Delete) для двух связанных сущностей: категорий и лекарств.

Основные возможности

Категории лекарств
- Просмотр списка категорий с пагинацией
- Добавление новой категории
- Удаление категории (только если в ней нет лекарств)

Лекарственные препараты
- Просмотр лекарств в выбранной категории с пагинацией
- Добавление нового лекарства (с валидацией всех полей)
- Редактирование любого поля существующего лекарства
- Удаление лекарства

Дополнительно
- Автоматическое создание структуры БД при инициализации
- Защита от SQL-инъекций (использование параметризованных запросов)
- Красивое табличное отображение данных (библиотека `tabulate`)
- Валидация вводимых данных (дозировка, объем, цена)

Технологии

- Язык: Python 3.x
- СУБД: PostgreSQL
- Основные библиотеки:
  - `psycopg2` — драйвер для работы с PostgreSQL
  - `tabulate` — форматирование табличного вывода
  - `PyYAML` — чтение конфигурационного файла

Установка и запуск

1. Установка зависимостей

Создайте файл `requirements.txt`:

psycopg2-binary
tabulate
pyyaml


Установка:

pip install -r requirements.txt


Или по отдельности:

pip install psycopg2-binary tabulate pyyaml


2. Настройка подключения к БД

Отредактируйте файл `config.yaml`:

dbname: lab7              # Имя вашей базы данных
user: postgres            # Имя пользователя PostgreSQL
password: your_password   # Ваш пароль
host: localhost           # Хост (localhost или IP)
dbtableprefix: "public."  # Схема БД (обычно public.)


Важно: Убедитесь, что база данных с указанным именем существует в PostgreSQL.

3. Запуск приложения

python main.py


Как пользоваться

При первом запуске
1. В главном меню выберите пункт 2 — это создаст таблицы и заполнит их тестовыми данными.
2. После инициализации выберите 1 для работы с категориями.

Главное меню

Добро пожаловать в систему учета лекарств!
Основное меню:
    1 - просмотр категорий лекарств;
    2 - сброс и инициализация таблиц;
    9 - выход.


Меню работы с категориями

Дальнейшие операции:
    0 - возврат в главное меню;
    3 - добавление новой категории;
    4 - удаление категории;
    5 - просмотр лекарств в категории;
    6 - следующая страница;
    7 - предыдущая страница;
    9 - выход.


Меню работы с лекарствами (внутри категории)

Дальнейшие операции:
    0 - возврат в главное меню;
    1 - возврат к списку категорий;
    6 - добавление нового лекарства;
    7 - удаление лекарства;
    8 - редактирование лекарства;
    9 - следующая страница;
    10 - предыдущая страница;
    11 - выход.

Валидация вводимых данных

|         Поле           |                     Требования                  |     Пример     |
|------------------------|-------------------------------------------------|----------------|
| Дозировка              | число + пробел + мг/мкг/млрд КОЕ (1-1000)       | `500 мг`       |
| Форма выпуска          | число + пробел + таблеток/капсул/флаконов/ампул | `10 таблеток`  |
| Цена                   | положительное целое число                       | `350`          |
| Рецептурность          | `да` или `нет`                                  | `да`           |
| Название/производитель | не может быть пустым                            | `Амоксициллин` |

Защита от SQL-инъекций: Приложение блокирует ввод SQL-ключевых слов (DELETE, INSERT, SELECT, DROP, UPDATE, ALTER, CREATE, TABLE).

Схема базы данных

Таблица `Categories`

| Поле        | Тип                  | Описание                     |
|-------------|----------------------|------------------------------|
| category_id | serial PRIMARY KEY   | ID категории (автоинкремент) |
| name        | text NOT NULL UNIQUE | Название категории           |

Таблица `Medicine`

|      Поле       |       Тип          |            Описание                    |
|-----------------|--------------------|----------------------------------------|
| medicine_id     | serial PRIMARY KEY | ID лекарства (автоинкремент)           |
| category_id     | integer NOT NULL   | Внешний ключ → Categories(category_id) |
| name            | text NOT NULL      | Название препарата                     |
| dosage          | text NOT NULL      | Дозировка                              |
| volume          | text NOT NULL      | Форма выпуска                          |
| producer        | text NOT NULL      | Производитель                          |
| is_prescription | boolean NOT NULL   | Рецептурный? (true/false)              |
| price           | integer NOT NULL   | Цена в рублях                          |

Ограничения:
- UNIQUE (name, dosage, volume, producer) — нельзя добавить два абсолютно одинаковых лекарства
- FOREIGN KEY (category_id) — при удалении категории с лекарствами операция блокируется

Возможные проблемы и решения

| Проблема | Решение |
|-------------------------------------------------------------|------------------------------------------------------------------------|
|   `ModuleNotFoundError: No module named 'yaml'`             | Установите PyYAML: `pip install pyyaml`                                |
| `ModuleNotFoundError: No module named 'psycopg2'`           | Установите psycopg2: `pip install psycopg2-binary`                     |
| `ModuleNotFoundError: No module named 'tabulate'`           | Установите tabulate: `pip install tabulate`                            |
| `OperationalError: FATAL: database "lab7" does not exist`   | Создайте базу данных в PostgreSQL: `CREATE DATABASE lab7;`             |
| `psycopg2.OperationalError: Connection refused`             | Проверьте, запущен ли PostgreSQL и правильный ли host в config.yaml    |
| Ошибка "Нельзя удалить категорию, в которой есть лекарства" | Сначала удалите все лекарства из этой категории через меню             |
| Ошибка "Ввод содержит запрещённые слова"                    | Не используйте SQL-команды в названиях и других полях                  |
